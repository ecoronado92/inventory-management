Factory Inventory Management System

Architecture overview — a full-stack demo application with a Vue 3 single-page frontend, a Python FastAPI backend, and in-memory mock data loaded from JSON files. No database.

Frontend: Vue 3 + Vite · port 3000 Backend: FastAPI + Uvicorn · port 8001 Data: JSON files, in-memory Tests: pytest (backend only)

System Architecture

Three tiers: a browser SPA, a stateless HTTP API, and JSON files read into memory at startup. The frontend calls the backend directly at http://localhost:8001/api (no Vite proxy); the backend allows it via permissive CORS.

Data Flow

Every page follows the same request cycle. Example: the Dashboard with filters set to warehouse=Tokyo, category=sensors, month=2025-03.

1FilterBar selectionFour global selects (period, warehouse, category, status) write singleton refs in useFilters.js
2View reactsEach view watch()es the filter refs and re-runs loadData() (also on mount)
3API clientapi.js builds URLSearchParams, omitting values equal to 'all', and issues axios GETs
4FastAPI routemain.py applies apply_filters() and filter_by_month() to the in-memory lists
5ValidationList endpoints validate through Pydantic response_models; summary endpoints return computed dicts
6Computed propertiesViews store raw responses in refs and derive chart/table data via computed()
7RenderKPI cards, SVG donut/bar charts, tables; currency formatted per locale (USD/JPY)

Notable Observations

Architecturally relevant gaps and characteristics worth knowing before extending the system.

  • Frontend/backend mismatches: api.js and the UI (TasksModal, Dashboard purchase-order buttons) reference /api/tasks (GET/POST/DELETE/PATCH) and /api/purchase-orders, but the backend implements none of these routes — the PurchaseOrder models and purchase_orders.json exist without endpoints.
  • Unrouted view: views/Backlog.vue is complete but not registered in the router; backlog data instead surfaces on the Dashboard.
  • Stateless, ephemeral data: all data is loaded into memory at backend startup; nothing persists between restarts and there are no write endpoints.
  • Direct cross-origin calls: no Vite proxy — the frontend hardcodes http://localhost:8001/api (and Reports.vue bypasses api.js entirely); the backend uses allow-all CORS, which is fine for a local demo but not production-grade.
  • Filter semantics: inventory has no time dimension, so the month filter only affects orders/dashboard; quarter values map to month lists via QUARTER_MAP.
  • Test coverage: backend endpoints are well covered by pytest (inventory, dashboard, misc endpoints); there is no frontend test setup.
  • i18n & currency: custom EN/JA translation layer with currency switching at a fixed USD→JPY rate of 150.
